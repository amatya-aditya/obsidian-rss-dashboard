### Coverage Gate (Ratchet)

CI runs `npm run test:unit -- --coverage` and enforces a global coverage floor via `vitest.config.mjs` thresholds.

Ratchet rules:

- Bump `lines`, `branches`, and `functions` by `+1` when the corresponding global metric is `>= (threshold + 0.75%)`.
- Never raise thresholds in the same PR that significantly expands coverage surface area (e.g., changing `coverage.include` to add `main.ts`).
- When bumping thresholds, record the new global snapshot in the PR description (or in a handoff doc) to keep the floor explainable.

Current thresholds (2026-03-31, post Phase 7): `lines: 40`, `branches: 33`, `functions: 34`.
