## 2.2.0

> RSS Dashboard 2.2 adds premium-grade filtering and highlighting, brings the plugin fully into its own on mobile and tablet, and delivers a major upgrade to reading, discovery, and feed management across the board.

---

### Headline Features

#### Highlight words and phrases like a power-user tool

- Highlight custom words or phrases across article titles, card summaries, and full reader content.
- Set per-word colors, choose exact or partial matching, and enable case sensitivity when you need precision.
- Control where highlights appear and quickly toggle them from the Filters menu.
- For heavy feed readers, this turns RSS Dashboard into a far more actionable scanning tool, not just a passive reading list.

#### Advanced include/exclude rules for serious feed curation

- Build global or per-feed rule sets using include/exclude logic.
- Target titles, summaries, or full content.
- Combine rules with AND/OR logic, override global rules on individual feeds, and bypass rules at the dashboard level when needed.
- This is one of the biggest workflow upgrades in 2.2: the plugin can now actively shape what you see, not just display everything a feed publishes.

### Full Mobile and Tablet Support

- 2.2 is the release where RSS Dashboard becomes genuinely comfortable to use across desktop, tablet, and phone.
- Android and iOS rendering issues were fixed across the sidebar, reader, icons, menus, and settings flows.
- Touch targets, drawers, reader controls, and settings layouts were reworked to behave more naturally on smaller screens.
- Mobile-safe reader settings, improved toolbar patterns, and responsive layout cleanup make the entire plugin feel far more first-class outside desktop Obsidian.

### Reading, Layout, and Sidebar Upgrades

- New Feed view adds a social-style, single-column reading mode with hero images and integrated actions.
- Reader formatting controls were added and refined for better readability across devices.
- Sidebar organization is far more flexible, with drag-and-drop ordering for feeds, folders, and toolbar icons.
- Tag filtering got a major upgrade with AND / OR / NOT logic and inline tag management.
- New options for unread-based sorting, hiding empty feeds, toolbar divider control, scoped sidebar search persistence, row spacing, badges, and toolbar visibility make the sidebar much more customizable.

### Discover, Feed Management, and Podcasts

- Discover got stronger with Kagi Smallweb integration, folder targeting, and an Add All button to quickly add feeds to your dashboard.
- Import and setup flows were improved with a more capable OPML experience, XML filename support, smarter default handling, and global feed defaults for new subscriptions.
- Feed management became more unified and polished across RSS, podcasts, and video feeds.
- Added native Twitter/X integration - simply paste a Twitter profile URL into the Add Feed window and it will be automatically converted into a chronological Nitter RSS feed.
- Podcasts received major quality-of-life upgrades including Pocket Casts support, improved feed resolution, sleep timer, better browser handoff options by extracting all URLs from the episode, and stronger metadata handling.

### Stability and Quality of Life

- Auto-refresh is smarter, including refresh-on-vault-open when an interval has already elapsed and an Off option for users who want manual control.
- Page-level mark read, auto-backup on unload, better pagination behavior, scroll restoration, and more reliable unread-state updates smooth out everyday use.
- Numerous fixes across reader rendering, OPML workflows, imported-feed defaults, proxy warnings, refresh behavior, and mobile regressions make 2.2 much more dependable than the beta line it grew out of.

### Under the Hood

- 2.2 also shipped with the first durable repo-wide automated test baseline for RSS Dashboard.
- The suite now sits at 120 passing test files and 1019 passing tests, with 54.66% statement coverage, 45.5% branch coverage, 49.58% function coverage, and 55.87% line coverage.
- Coverage thresholds are actively enforced in `vitest.config.mjs` at lines 40, branches 33, and functions 34 to help prevent regression.
- Under the hood, this release included major maintainability work across settings architecture, feed manager flows, reader components, article list extraction, shared test infrastructure, and general release hardening.
