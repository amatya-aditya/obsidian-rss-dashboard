### 🔄 Latest 2.2.0 Updates (Pre-Commit)

### ⚠️ Important: Database Migration (for Existing Users)

Starting in **2.2.0**, feed data is migrated from JSON storage to SQLite for better performance.

- **What migrates:** feeds, articles, folders, and tags
- **One-time migration:** runs automatically on first launch after updating to 2.2.0
- **What to expect:** first load may take longer than usual if you have a large feed/article history
- **Backup created:** `data.json.backup` is written in the plugin folder during migration
- **New storage split:**
  - `rss-dashboard.sqlite` stores bulk feed/article data
  - `usersettings.json` stores plugin settings

If something looks wrong after migration:

1. Run command palette action: **Restore from JSON backup**
2. Reload Obsidian/plugin

Fresh installs are not affected by this migration path.


#### SQLite Reliability & Recovery

- Added SQLite integrity checking on open; unreadable databases are detected and reset safely
- Added automatic fallback recovery from `data.json.backup` / `data.json` when SQLite corruption is detected
- Enhanced JSON import flow: if a selected JSON file includes feeds/folders/tags, it now restores full data (not settings-only)
- Hardened **Restore from JSON backup** with validation, DB re-init safeguards, and persistence to both SQLite + `usersettings.json`

#### Database Maintenance Workflow

- Added new command palette action: **Optimize database storage**
- Added storage maintenance routine using checkpoint + `VACUUM` + `PRAGMA optimize`
- Added conditional auto-compaction after **Delete all feeds** for large purges or large DB file sizes

#### Large OPML Import Performance & UI Responsiveness

- For very large OPML imports, immediate full SQLite sync is skipped to avoid blocking import startup
- Background import now scales save/render frequency based on queue size
- Full SQLite sync is deferred while background import is active, then executed once at completion
- Sidebar import-state lookup was optimized to avoid O(N²) queue scans during massive imports
- Feed rows remain more responsive/clickable during long-running imports

#### Reader Experience

- Improved reader fallback: when full-page extraction is weak/empty, reader now falls back to feed-provided HTML (`content`/`description`)
- Added a collapsible **Feed description** callout block in Reader view (expanded by default)
- Removed the separate top cover image block so reader content starts immediately with article text

#### Sidebar Search Workflow

- Sidebar feed search query now persists across sidebar re-renders
- After clicking a searched feed/folder, search results remain filtered so you can open matches one by one without retyping
