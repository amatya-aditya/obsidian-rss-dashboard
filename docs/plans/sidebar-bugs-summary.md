# Sidebar Bugs Summary & Action Plan

This document summarizes the three current bugs affecting the RSS Dashboard sidebar and provides a breakdown of the root cause and proposed fix for each.

## 1. Folder & Subfolder Show/Hide Bug

**Issue:** Certain folders—specifically the last folder in the list as well as subfolders inside other parent folders—fail to properly show or hide their child elements when toggled.
**Root Cause Assessment:**

- _Previous Theory (DISPROVED):_ Initially, it was believed that this was an optical illusion caused by the sidebar scroll viewport shifting upwards when the last folder collapsed, causing newly expanded items to render below the fold.
- _Current Reality:_ The user confirmed the children are actually failing to show/hide, and the bug also impacts subfolders independent of their position in the list. The root cause is likely a logic failure in how the `.collapsed` class is applied, or an issue with the CSS max-height transitions not propagating correctly to nested elements.
  **Attempted Fixes:**
- _Auto-Scroll Logic:_ Implemented a `setTimeout` function that calculated whether a newly expanded folder fell below the visible viewport, to smoothly scroll it into view. This was unsuccessful as the elements were genuinely not rendering/displaying correctly, not just hidden by scroll.
  **Next Steps:**
- Re-evaluate the `handleToggleFolderCollapse` and `renderFolder` logic to determine why specific folders or subfolders might be resisting the collapse/expand state change.
- Verify `max-height` and `overflow` CSS properties on nested element layers.

## 2. Subfolder Indentation Bug

**Issue:** Subfolders do not receive the proper visual indentation relative to their parent folder, despite the custom row indentation settings.
**Root Cause:** In the DOM structure generated by `sidebar.ts`, subfolders (`.rss-dashboard-feed-folder`) are appended as direct _siblings_ to their parent's feed container (`.rss-dashboard-folder-feeds`), instead of being nested _inside_ the feed container. Because the custom CSS indentation variable (`var(--sidebar-row-indentation)`) is only applied to the `.rss-dashboard-folder-feeds` container, the sibling subfolders don't inherit the parent's padding recursively.
**Potential Fix:**

- Refactor `renderFolder()` so that subfolder elements are appended _inside_ the parent's `folderFeedsList` alongside the feed items. This will naturally compound the `padding-left` for each level of nesting, creating perfect cascading tree indentation automatically. This also simplifies the collapse logic, as hiding the parent's feed container will properly hide all its nested subfolders without complex DOM queries.

## 3. Parent/Child Folder Drag-and-Drop Bug

**Issue:** Feeds cannot be placed inside a parent folder if a child folder exists inside the parent.
**Root Cause:** The drop zones in the sidebar are strictly defined on the folder headers (`.rss-dashboard-feed-folder-header`). The main empty space and the actual list of feeds within a folder are handled by a fallback listener that assigns dropped feeds to the "Root" (no folder). Because subfolders split up the parent folder's visual space, users intuitively try to drop feeds onto the parent's list of child feeds, which misses the header row entirely and ends up hitting the fallback Root drop zone.
**Potential Fix:**

- Add drop event listeners to the `.rss-dashboard-folder-feeds` container as well. If a feed is dropped into the area showing a parent's children (or dropped between its subfolders), the drop handler should identify which parent folder container it was dropped into and assign the feed to that path.
