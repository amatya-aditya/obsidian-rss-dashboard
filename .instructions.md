# RSS Dashboard AI Contribution Instructions

Last updated: 2026-05-13

This file is a quick policy card for AI-assisted edits.

## Compliance Declarations (Required)

1. Never add unsafe HTML injection in production paths. Do not use direct `innerHTML` for feed/article rendering; use approved sanitizing append utilities.
2. Do not add `eslint-disable` without an inline reason. Prefer refactoring over suppression.
3. Avoid `as any` sprawl. Use one boundary cast (`as unknown as TypedInterface`) and keep the rest strongly typed.
4. In popout-sensitive UI code, prefer `activeDocument` and `activeWindow.setTimeout/clearTimeout` over global `document` and timers.
5. Follow Obsidian DOM helper conventions in production code (`createDiv`, `createEl`, `createSpan`, `createFragment`) unless a documented exception applies.
6. Follow existing restricted-import guidance and approved API alternatives.

## Pre-PR Checks

- Run `npm run lint`.
- Run `npm run test:unit` (or targeted tests with rationale).
- Confirm no new undocumented suppressions.
